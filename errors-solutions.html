CSE 340 - Errors and Solutions

When using XAMPP on Windows you get the error code api-ms-win-crt-runtime and it stops Apache from executing.
A Google search turns up these possible solutions
  • https://www.youtube.com/watch?v=EgixgR_OW98
  • https://softwarekeep.com/help-center/how-to-fix-the-api-ms-win-crt-runtime-l1-1-0dll-is-missing-error-in-windows
    • Using Docker, in my docker-compose.yml file, I missed the 's' in "phpmotors" when I was setting up my Docker/phpmyadmin. To be more clear, my line 43 in the docker-compose.yml said "MYSQL_DATABASE: "phpmotor"" instead of "MYSQL_DATABASE: "phpmotors"". I continued my set up of Docker, and I don't have a phpmotors database. I tried to delete/restart my containers after I fixed the error, but I fear I may have to set up a completely different virtual environment and folder.
    The quick solution is to stop all of the containers then delete them. You can use the Docker Dashboard for this. Then, delete the /cse340/dbdata/ folder, then restart the Docker containers.

Week 3

This week you‚Äôll learn how PHP and MySQL talk to each other. You will create a database and add some tables. You can think of a database like a collection of spreadsheets. Each spreadsheet is like a database table. When you create a table, it‚Äôs like you hard code the title of each column, and hard code what type of data (numbers, letters, or any character) can be put into that column.

Each row of the spreadsheet is a record in the database table. When you add stuff to a database, you are creating records, or rows in a spreadsheet.

You will also create a database user. This is a user account (username and password) to access the database. Your website will need these credentials in order to connect to the database and Create, Read, Update and Delete (the CRUD).

You‚Äôll put the username, password and database name into a php file that will hold all of the details for connecting to the right database.

Week 4

In this week‚Äôs activities, you‚Äôll learn to use the MVC structure. This is basically just a way to break up our code into manageable chunks. It‚Äôs kind of like separating content to HTML and style to CSS. In MVC you are separating out the database parts (the Model), the HTML displayed in the browser (the View), and the code that decides which view to show and which data to pull from the database (the Controller).

You‚Äôll use data from the database to build the navigation. The data from the database will come to the PHP file in the form of an array. We can then loop over that array to build the navigation links.

This brings us to the magic of the URLs. We‚Äôll be using the URLs to tell the code which page to display. The main index.php page (the main controller for the site) will have code (a switch statement) that checks the URL for a variable named ‚Äúaction‚Äù and the value of that variable, which will determine which page to display. The URL might look like http://localhost/?action=login the ‚Äú?‚Äù tells PHP that a variable and its value are to follow. In this case, the variable is ‚Äúaction‚Äù, the value is ‚Äúlogin‚Äù. We‚Äôll be using this pattern throughout the site to determine which page (view) to display.

Week 8

For the activity part of this week, you‚Äôll learn about web cookies and how to create one for making a personalized login screen.

A good thing to keep in mind here is that sessions and cookies are just bits of information (think of it like a text file) that is kept by your web browser. Very generally speaking, the web server sends a bit of information to the browser and the browser keeps it in a special spot until the browser visits that site again. Then the website can do lots of cool stuff like put up a personalized welcome message, keep track of certain preferences, and so forth.

You‚Äôll also learn about creating PHP sessions. Sessions are like an open communication channel between the server and the web browser of whoever is visiting your site. You will use the session to facilitate the login and log out process. You‚Äôll also create the admin page to manage users and access levels.

We‚Äôre turning this into a real content management system! üòä

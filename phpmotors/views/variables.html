PHP Variables

What is a variable?

A variable, in PHP, is a container to hold a value. As the name implies, the value stored in the variable can change (or vary) from one line of code to the next.

Variables allow you to store data or values and then use them whenever needed without having to specifically write the exact value. Consider a login form, where the client is asked to provide a username and password.

Every time the form is filled in by a different client, the data will be different. I may use a username and password of "BarneyOne" and "Purple1sK1nG" while you may use "HeMan" and "Mast3rUn1vers3". We need a simple way of working with whatever values the client enters. The way to do this is by using variables.

Most of the time variables are declared by the programmer. Using the previous example, two variables called $username and $password, could be created and used to store any input values from the login form.

Using variables and displaying their values will be shown later, but the important point now is to understand how the use of variables can make handling data easier.

Creating Variables

In PHP, simply writing the name of a variable for the first time in a script creates it. However, there are a few rules to remember:

  1. The name starts with a $ sign
  2. The first character after the $ must be a letter or underscore
  3. The following characters can be a combination of letters, numbers and/or underscores
  4. Variable names are case sensitive

$username is a valid name because it meets all three rules, while $55skidoo is not valid because it starts with a number â€” remember, the first character after the $ sign must be a letter or underscore.

Also, variable names should be meaningful. If the data you will be storing is a user name, a sensible name might be $username. You could also call it $zelda, but that doesn't really describe what value it is holding.

When writing variable names it is important to be consistent in all of your code. For example, you can use camelCase to combine compound names (e.g. $userName, or use an underscore (e.g. $user_name), or a run-on name (e.g. $username). All three are fine, just choose one and use it consistently. It is important to remember that PHP treats $username and $userName as completely different!

Assigning Variables

Now that you can create a variable, let's assign a value to one.

$userName = "HeMan";
$password = 'Mast3rUn1vers3';
$age = 21;

First, the variable $userName is given the value 'HeMan'. This is referred to as assigning a value. And because this is the first time $userName is used, the variable is created automatically. Anytime you write $userName after that, PHP will know to use the value 'HeMan' instead.

Then, $password is written and assigned the value 'Mast3rUn1vers3'. A variable can be created without assigning a value to it, though this is generally not considered to be good practice. It is better to assign a default value just so you know it has a value. Remember, variables are variable, so you can always change the value later.

Notice how the types of data referenced by a variable have different types. This property is called the data's data type. 'HeMan' and 'Mast3rUn1vers3' are wrapped in quotation marks, so they are both strings (string is just another name for text). 21 is obviously a number (more specifically, an integer).

Here are some variables with different data types:

$net = 0; // Integer
$title = "PHP Variables"; // String
$flag = TRUE; // Boolean
$amount = 99.99; // Double
$pets = array(Dog, Cat, Fish, Mouse); // Array

Now that you understand the basics of naming variables and assigning values lets restate what has already been demonstrated. The examples above show that values to the right of the = sign are assigned to the variable name on the left of the = sign. This is always the case: values on the right are assigned (or stored in) the variables on the left.

Displaying the Value of Variables

As you saw at the beginning, you can display the value represented by a variable using echo. Another option is to use print. For all intents and purposes they are equivalent, so pick one and use it.

echo $userName;

Perhaps you would like to add additional text prior to displaying the value of the variable (this is commonly done, particularly when troubleshooting). For example:

echo "The user name is: " . $userName;

The dot between the text in quotation marks and the variable name is the concatenation operator. It joins the string on the left and the value of the variable on the right together.

Double vs. Single Quotes

You could avoid using concatenation and make use of interpolation instead. Interpolation is when the variable name appears in a string and is replaced by its value instead. However interpolation only happens when variables are within double-quotation marks! If you enclose a variable in single-quotation mark you get exactly what is in the quotes - the name of the variable, not its value. This is because any string in single quotes is called a string literal, which as the name implies is literally what is in the string.

echo "User name: $userName";
// Result: User name: HeMan

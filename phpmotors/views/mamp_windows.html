MAMP for Windows

Introduction

An ideal situation for web development is to have a local environment for development and testing and a remote production environment for distributing your work to the rest of the world. To create a local environment requires the installation of server software on your own machine. For web development using PHP, this is known as an "AMP" (Apache, MySQL and PHP) stack. This software bundle gives your machine many of the same capabilities as a dedicated server. On Windows, the current bundle discussed in this tutorial is MAMP (Apache, MySQL and PHP) for Windows.

This document covers five topics:

  • Download Mamp
  • Configure Mamp
  • Using the Start Page tools
  • Enable Error Display
  • Troubleshooting

Download MAMP

Video Demonstration

This video demonstrates downloading MAMP for Windows.

Using your web browser go to mamp.info. Once there, find and download the latest version of MAMP by clicking the "Free Download" button next to the MAMP icon on the left (Not the MAMP PRO icon).

When the download page opens click the option on the right for Windows. The file will download to your computer.

Installation

  1. Locate and launch the downloaded installation file on your computer by right-clicking the file and selecting "Run as Administrator".
  2. If presented with the security warning(s) asking if it is OK for the software to change your PC?, proceed by clicking "Yes".
  3. You will also be warned about your User Account Control not allowing for installation to the Program Files. Click "OK".
  4. When the install begins you will shown a list of software to install. Turn OFF the checkboxes to install MAMP PRO and Apple Bonjour.
  5. Accept the license agreement.
  6. On the following screen accept the default installation location.
  7. On the following screens accept the default options until the installation finishes. Note: When the progress bar reaches the 98% mark there will be a noticeable pause. Just let it run, it will eventually finish.
  8. When done, click "Finish" to close the installer.
  9. There should now be a shortcut icon on the desktop that can be used to launch MAMP. I recommend you move it to the taskbar for ease of access.

Configuring MAMP

Video Demonstration

Cloud Drive

Prior to setting up MAMP you will need a cloud drive account and the app for your drive should be installed and working on your computer. Google Drive, Dropbox and OneDrive are all examples of cloud drives. By having a cloud drive, your data is automatically backed up to the drive and should your computer harddrive fail, you will not lose your data.

Create the Web Root Folder

In the cloud drive, on you local computer, create a folder named "htdocs". Make sure it is all lower case! This will become the root folder for your web site and where all work for the class will be done.

Starting Apache and the Database servers in MAMP

  1. Launch MAMP using the desktop icon, the taskbar icon (if you created one), or the start menu.
  2. The control panel has a start servers button which should activate as soon as MAMP is run.
  3. Both servers (Apache and MySQL) should start automatically. If they do not, consult the Troubleshooting section.
  4. When the process starts the circle icon to the right of the text will turn green.
  5. When both icons are green, click the "Open WebStart page" button and a browser window should open showing the localhost (your computer) start page.
  6. If you see the start page, the server is installed and running as it should.
  7. If you encounter an error, consult the "Troubleshooting" section below.

Running and Stopping the MAMP Servers

Running servers on your local machine is a useful tool, but one that can potentially use much of your computer's resources. Therefore, it is important to start MAMP only when you are developing and testing using PHP and shut it down when finished. We will change a few setting to make this easier.

Setting the Servers to Stop Automatically

  1. In the MAMP control panel, click the "Mamp > Preferences" menu option.
  2. Make sure the "Start/Stop" tab is active.
  3. Turn ON the checkbox to "Stop Servers when quitting MAMP.

Designate the Root Folder

  1. Click the "Web Server" tab.
  2. Click the "Select..." button to the right of the text box.
  3. Navigate to the "htdocs" folder in your cloud drive and select it.
  4. When returned to the MAMP preferences window, make sure the path looks correct.
  5. Click "OK" to close the preferences window.
  6. The servers should automatically restart to implement the changes.

Test That It Worked

Click the "Open WebStart Page" button. A new web page should open in your default browser. If it does, then things are configured and working. We'll explore this start page in the next section.

phpMyAdmin

phpMyAdmin is a database management tool and the primary tool that we'll access from the Start page. To access it, select "Tools > phpMyAdmin" from the navigation bar.

The tool's window will open. We'll spend much more time learning and using this tool in future lessons. For now there are two things to remember:

  2. Never, Ever change any of the four existing databases shown on the left of the screen. These are critical to the basic operation of the database server and should be left alone!

Displaying Errors

By default MAMP hides all errors that it encounters. While this is good in a production environment it is not good for our development environment. Therefore, we need to enable the displaying of errors to help us as we develop our application.

  1. Open phpInfo as done earlier.
  2. Scroll down until you find the item "Loaded Configuration File". Copy the path shown to the right of the title.
  3. Open your file explorer and paste the path into the address bar at the top of the window. Press Enter.
  4. Open the file in any text editor.
  5. Do Contol + F to open the search tool.
  6. Search for "display_errors".
  7. Click the "Find Next" button to move down through the page until you find the line that says, display_errors = off (the line will not have a semi-colon at the far left and should be about 25% down from the top of the document).
  8. Change the word "off" to "on".
  9. Save and close the file.
  10. Return to the MAMP control panel and stop, then restart the servers.
  11. From now on, if PHP encounters an error, it will report the error to you at run-time.
  12. You should only have to do this once.

Troubleshooting

Use the following troubleshooting items if the Apache or Database servers will not start

Turn Off Conflicting Programs

If the Apache server will not start it may be due to others programs running that are using the web port. Check if any of the following are running:

  • Skype - Quit it.
  • VMWare — Use this Error Correction PDF to turn it off or change settings.
  • Conflicting services - Open the "Services" window and look for any services that are using http, httpd or mysql - Windows Services Manager. See the "Check the Task Manager" section below.
  • Windows 10 has a service named "World Wide Web Publishing Service" that uses port 80, wit a PID of 4. If it is running the Apache server will not start. Use the Task Manager to stop the service.

Check the Task Manager

  1. Normally the Apache server should start without issues.
  2. However, if it will not start it is typically because another web server is already running and will have to be stopped.
  3. Launch the Windows Task Manager (Control + Alt + Delete keys held down together).
  4. Click the Task Manager link.
  5. In the Task Manager, click the "Processess" tab and look in the "Background Processes" list.
  6. If you see an "Apache HTTP Server" in the list click it, then click the "End Task" button until that process disappears.
  7. When it is removed from the list, return to the XAMPP Control Panel and click the "Start" button for Apache.
  8. It should start and the background for the Apache text should turn green.

Conclusion

With MAMP installed and configured you're ready to begin your work. If for any reason you cannot get the servers to start, get help immediately. Don't put it off. Without the servers you will not be able to proceed in our class.

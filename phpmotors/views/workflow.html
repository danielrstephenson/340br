Development Workflow

In a live workflow, materials would be developed and tested locally and posted to a production server. However, for this class, we will work entirely in a local development environment and not make the developed resources available to the rest of the world. Instead, you are to follow this process:

  1. Build all resources within the acme folder within the htdocs folder.
  2. Create a meaningful directory (folder) structure within htdocs (e.g. css, images, modules, homework, project - while following the web naming rules)
  3. Place all files (as they are built) inside of the appropriate directory(s).
  4. Develop the solution as needed and test frequently until operational.
  5. When preparing to submit work:
    1. Create a copy of the assessment folder.
    2. Zip the folder.
    3. Rename the zip file to reflect the name of the assessment.
    4. Store the zip file to retrieve later if needed.
    5. Delete the copy of the htdocs folder.
  6. Make a video of your solution working in your development environment. The video must be narrated and explain how the application does what it should. The video should be concise and last no longer than 4 minutes.

    7. Solution Demo

    The following solution is intentionally not fully functional. It is intended as as demo only for the sake of the video demonstration.

  7. Post your video to your YouTube channel and capture the share link to the video.
  8. Submit Your Work:
    1. Go to the appropriate Assignment dropbox in I-Learn.
    2. Click the dropbox link to open the submission screen.
    3. Paste the assessment video link into the comments box.
    4. Click the "Add A File" button.
    5. Drag the assessment zip file to the drop zone in the pop-up window.
    6. Click the "Add" button (bottom left corner of the pop-up window.
    7. Click the "Submit" button (bottom left corner of the assignment window.

Creating a Proxy User

What is a proxy user?

When visitors come to your dynamic web site they want to see your content. Being a dynamic web site, that content is stored in a database. A proxy user represents (is a proxy for) a site visitor and interacts with the database in their behalf.

Limited Capabilities

The proxy user is generally only given "data manipulation" privileges. This means the proxy can read (SELECT) data, add (INSERT) new data, change (UPDATE) data, sometimes upload files (FILES), and perhaps remove (DELETE) data from the database. However, the proxy is never given the ability to change the actual database in any way!

Creating a Proxy Database User

For Docker:

The process of creating the MySQL server with Docker creates a proxy user for us. The username and password are in the docker-compose.yml file. As a reminder, that information is

        MYSQL_USER: "dbuser"
        MYSQL_PASSWORD: "dbpass"

It is not necessary for you to create a new proxy user, but this is good practice. In order to create a new MySQL user, follow the steps in the video below.

In order to create a user, you will need to log in to phpMyAdmin as the root user (unless you changed them in the docker-compose.yml file).

        username: root
        password: rootPASS

As you follow the video instructions, use a percent sign % instead of localhost for the "Host Name". This is wildcard variable, meaning any host is allowed.

Enhancement 9

Overview

In this enhancement you are asked to implement enhanced functionality for the site based on image uploads. This enhancement will consist of three general tasks:

  1. Run the current vehicle images through the new image upload tool, which will move them to a new location, and add their file path to the database.
  2. Revise some of the work that was done last week in order to get the vehicle details from the inventory table, but the path for the large, primary image from the images table.
  3. Add new functionality so that we can add vehicle thumbnails to the vehicle detail page and style everything to look professional.

Video Overview

This short video demonstrates what the finished enhancement will look like and the functionality it should possess.

Tasks

Add Vehicle Images

  1. Change the target of image uploads in the uploads controller to the "vehicles" folder within the phpmotors "images" folder from the current "uploads > images" folder.
  2. Delete all records from the images table so that it is empty.
  3. Make a duplicate of the vehicles folder to your desktop. Once done, delete all of the images inside the images/vehicles folder in the PHP Motors site.
  4. Upload all existing vehicle images using the upload process so that the images will be sized correctly and stored into the "images" table of the phpmotors database and to the "images > vehicles" folder. Each of these uploads should be designated as "Primary" images.
  5. Add the DMC DeLorean to the inventory table.
  6. Upload the DeLorean image and associate it with the car, making it the primary image.
  7. Associate the "no-image" picture to the "Dog Car" in the inventory table. Make it the primary image for that vehicle.
  8. Add at least one additional image to three vehicles. These images will NOT be primary images, rather they will be additional images for the vehicle.

Revisions

  1. Alter the code for the home view so that the large, primary DeLorean image is now displayed in the home view from its location in the "phpmotors/images/vehicles" folder.
  2. When done uploading the images you will need to alter the functionality for displaying vehicles by classification. All images should now be obtained from the images table and not the inventory table. In the model it will require the use of a "JOIN" and a "LIKE" in the in the query, as well as limiting images to primary, thumbnail images. When done, click each classification item in the navigation bar and make sure the primary thumbnail image of each vehicle appears in the classification view and is from the images table.
  3. Revise the function for obtaining an individual vehicle's details to display in the detail view so that the vehicle details is still obtained from the inventory table, but the primary, large image comes from the images table. This function should only obtain the large, primary image from the images table, and the non-image details from the inventory table.
  4. Do NOT alter the function that obtains vehicle information that is used in the vehicle editing process.

New Functionality

  1. When the "vehicle detail" view is loaded the large, primary image should display (again, from the images table). But what if additional images have been uploaded for that vehicle? It would be beneficial if "thumbnails" of all images pertaining to that item were displayed. Therefore:
    1. In the uploads model, build a new function that will obtain thumbnail image information from the "images" table that are based on the vehicleId. You will need to use the LIKE operator in your SQL query to find "-tn" in the file name.
    2. In the vehicles controller, call the function when the vehicle detail page is requested. The uploads model will have to be brought into the controller's scope.
    3. The returned list of thumbnail images should be fed into a custom function that will wrap them in HTML so they are ready to be viewed. No additional information about the image needs to be included, just the thumbnail image itself with an "alt" value added to the image.
    4. The HTML containing the images should be included into the "vehicle detail" view by the "vehicle" controller when the view is delivered and styled to be professional in appearance. The thumbnails must appear on the left of the vehicle detail information on large screens, and below the information on mobile screens. On mobile screens a heading should appear above the thumbnail images.
    5. Add CSS that will cause a thumbnail to enlarge in size when hovered over. When the mouse leaves the thumbnail image, it should return to normal size. This should occur only on large screens, not mobile devices.
    6. Add at least one additional non-primary image (it doesn't matter if they actually match the vehicle) to at least three inventory vehicles.
    7. To test, access the "vehicle detail" view via the car classification view. Make sure the additional thumbnail images appear and are professional in their appearance. The view should continue to be valid HTML5 and valid CSS3 when done.

Grading Matrix

This enhancement has values in multiple objectives as shown below:

Objective 1

  • The vehicle detail view is valid HTML and CSS, and is responsive.
  • Full size images are no larger than 500 pixels wide or tall
  • Thumbnail images are no larger than 200 pixels wide or tall
  • Total value for Objective 1: 10 points

Objective 2

  • A new custom function exists in the functions.php file to wrap the vehicle thumbnail images in HTML and return the result to the vehicles controller.

  • Total value for Objective 2: 10 points

Objective 4

  • A function to extract thumbnail images related to a vehicle exists, uses a PDO Prepared Statement approach and works.
  • Total value for Objective 4: 5 points
